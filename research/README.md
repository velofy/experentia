# experentia research layer

The evidence base under the taste-engine. Built 2026-07-09 by a deep-research harness: 5 search angles, 24 sources fetched and graded, 118 claims extracted, the top 25 put through 3-vote adversarial verification panels (2 of 3 refutes kill a claim).

Result: **19 claims confirmed, 0 refuted, 6 verification-errored** (an API session limit interrupted the style-stream verifiers and the synthesis pass; re-verification is queued and this layer will be updated in place).

## Files

| File | What it is |
| --- | --- |
| `claims-verified.json` | The machine layer: every confirmed and pending claim with source, quote, and vote record, plus the 24 graded sources. |
| `claims-raw.json` | All 118 extracted claims grouped per source document, before verification. Working knowledge, not citations. |
| `color.md` | Color doctrine: ecological valence theory, blue/trust, chartreuse avoidance, cultural caveats, WCAG floors, with engine implications. |
| `style.md` | Style doctrine: 50ms first impressions, complexity and prototypicality, the aesthetic-usability effect (pending), with engine implications. |
| `patterns.md` | Navigation, loading-state, and performance patterns from NN/g, web.dev, and the skeleton-screen literature, plus famous-site convergences. |

## Reading order for an agent

1. `color.md` and `style.md` for the why.
2. `patterns.md` for the how.
3. `../taste/` for the distilled machine rules (generated from this layer).
4. `../checklists/pre-ship.md` before shipping anything.

## Honesty rules

- A claim's verification status is always carried with it. Unverified means unverified.
- Where the evidence conflicts (skeletons vs spinners), the conflict is recorded, and the engine rule explains how it navigates the conflict.
- This layer cites what it uses; a rule with no citation is experience, and says so.
