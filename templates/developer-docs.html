Quickstart

This guide takes a new project from an empty folder to a running local server in about five minutes. Every command below is copyable, and every heading links to itself, so you can share an exact spot in the page.

Install the toolkit #

Install the packages with your usual package manager. The install line pins exact versions, so a fresh checkout resolves the same dependency tree every time.

bash 
$ npm install @example/ui-kit @example/tokens @example/icons --save-exact --registry=https://registry.example.com/npm/

Prefer to keep a lockfile in source control. If you use a different client, swap npm install for the equivalent, for example pnpm add or yarn add.

Initialize a project #

Run the initializer from the folder you want to use as the project root. It writes a config file and a starter entry point, and it never overwrites files you already have.

Create the config #

The initializer drops an example.config.js at the root. Open it and set the source and output folders to match your layout.

example.config.js 
// example.config.js
export default {
  root: "./src",
  output: "./dist",
  tokens: "@example/tokens",
  strict: true,
}
Note

The config is plain JavaScript, so you can compute values at load time. Keep it free of secrets, and read credentials from the environment instead. See the Configuration guide for every option.

Run the dev server #

Start the local server. It watches the source folder and reloads the page whenever a file changes.

bash 
$ npm run dev -- --host 0.0.0.0 --port 4173 --open

The server prints a local URL and a network URL. Open either one in a browser to see the starter page.

Warning

Binding to 0.0.0.0 exposes the dev server to your local network. Use it only on networks you trust, and never as a production server.

Next steps #

Once the server is running, keep going:

  • Read Core concepts to learn how the pieces fit together.
  • Follow the Configuration guide for the full option reference.
  • Set up Authentication before you deploy.
